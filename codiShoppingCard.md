# ¿Qué es una Pull Request?

## Definición
Una Pull Request (PR) es una funcionalidad de Git que permite solicitar la fusión de cambios de una rama a otra.

## Para qué sirve
- Solicitar revisión de código
- Discutir implementaciones
- Integrar cambios de forma controlada
- Mantener historial organizado

## Ventajas ✅
1. **Control de calidad:** Múltiples ojos revisan el código
2. **Documentación:** Cada cambio queda registrado
3. **Colaboración:** Facilita trabajo en equipo
4. **CI/CD:** Puede desencadenar tests automáticos

## Desventajas ❌
1. **Tiempo:** Proceso puede ser lento
2. **Conflictos:** Si varias personas modifican lo mismo
3. **Aprendizaje:** Curva inicial para nuevos usuarios

## Proceso típico
1. Crear rama desde main
2. Realizar cambios
3. Hacer commits
4. Push al repositorio
5. Crear PR en GitHub
6. Revisión y comentarios
7. Aprobación y merge
8. Eliminar rama
